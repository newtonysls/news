# Agentic Retriever 设计说明

这个文档对应两个核心目标：

1. **黑盒搜索 -> 可导航搜索**：让 Agent 不只是「抽卡式」召回，而是能显式选择知识库位置并查看。
2. **迭代学习 -> 路径复用**：将成功检索路径做总结，后续类似问题优先走捷径。

## 1. 系统模块

- **Knowledge Navigator（知识导航器）**
  - 把知识库抽象为可定位的节点（章节、文档、段落、表格等）。
  - 提供三个基本动作：
    1) `inspect(node_id)`：查看某个节点内容。
    2) `jump_candidates(hint)`：按 hint（问题、关键词、大纲线索）跳转候选节点。
    3) `neighbors(node_id)`：扩展当前节点的局部邻域。

- **Retriever Agent（检索代理）**
  - 输入问题后执行“观察 -> 决策 -> 动作”的闭环。
  - 决策依据来自：
    1) 用户问题。
    2) 当前查看到的节点内容。
    3) 历史 memory 的路径经验。

- **Summary + Memory（总结与记忆）**
  - 每次完成检索后，将路径压缩为可复用规则：
    - 触发模式（pattern）
    - 起跳点（first_hop）
    - 成功路径（A -> B -> C）
    - 路径理由（rationale）
  - 下次查询先检索 memory，若匹配则直接尝试捷径。

- **Markdown KB Loader（知识库入口）**
  - 支持将 Markdown 文件解析为节点（按 heading 切段）。
  - 默认邻接关系为同文档上下节互连，可用于模拟“人工翻页式”查找。

## 2. 关键机制

### 2.1 可导航检索（解决黑盒问题）

传统向量检索只有 `query -> topk chunks`，过程不透明。

这里增加“可导航”能力：

- 先粗跳转到可能章节（例如通过目录、标题匹配、一次向量结果反推位置）。
- 再在局部邻域做迭代扩展（近邻段落、上下游引用、关联实体页）。
- 每一步都记录 `为什么跳到这里`，提升可解释性。

### 2.2 经验压缩与路径捷径（解决迭代时延）

对于很多重复问题，首次路径可能是 A -> B -> C，但后续可以直接 A -> C。

实现上不依赖训练：

- 使用总结器把历史成功轨迹存入 memory。
- 查询时做“模式匹配”（关键词/语义相似）找到历史轨迹。
- 命中时优先执行捷径路径，降低检索步数与时延。

### 2.3 可观测检索路径（可测试效果）

命令行测试会打印：

- `visited_path`：显式路径（例如 `doc::0 -> doc::1 -> doc::2`）。
- `reasoning`：每一步跳转和停止原因。
- `used_memory`：是否命中历史经验路径。

这使得你可以直接评估“首次探索 vs 后续复用”的路径差异。

## 3. 代码原型

见 `agentic_retriever.py`：

- `parse_markdown_file` / `load_markdown_knowledge_base`：将 Markdown 作为知识库入口。
- `KnowledgeNavigator`：透明导航能力。
- `RetrievalMemory`：保存路径经验。
- `AgenticRetriever`：先 memory 快捷路径，不命中再做探索。
- CLI (`run_cli`)：可直接测试检索效果与路径。

## 4. 线上化建议

- 将 `DocNode` 扩展为多粒度节点：文档/章节/段落/句子。
- `jump_candidates` 可接入向量数据库 + BM25 混合召回。
- Memory 增加 TTL、版本、数据域标签，避免错误经验污染。
- 总结器可由小模型执行，将冗长轨迹压缩为结构化 JSON。
- 增加评估：
  - 命中率（answer hit）
  - 平均路径长度（steps）
  - 平均时延（latency）
  - 路径可解释性（人工打分）
